# 메모리의 이해

## 메모리란?
**메모리(Memory)** 는 CPU가 처리할 데이터와 명령어를 **임시로 저장**하는 공간입니다.
프로그램은 실행되기 위해 반드시 메모리에 올라와야 합니다.

---

## 메모리 계층 구조
속도와 용량은 트레이드오프 관계입니다. (빠를수록 비싸고 용량이 작음)

```
         빠름 / 비쌈 / 용량 작음
              ┌───────────┐
              │  레지스터  │  ← CPU 내부 (수십 바이트)
              ├───────────┤
              │캐시(L1/L2)│  ← CPU 내부 (수KB~수MB)
              ├───────────┤
              │    RAM    │  ← 메인 메모리 (수GB)
              ├───────────┤
              │ SSD / HDD │  ← 보조 저장장치 (수TB)
              └───────────┘
         느림 / 저렴 / 용량 큼
```

---

## 주소 공간 (Address Space)

### 물리 주소 (Physical Address)
- 실제 RAM에 존재하는 주소
- 하드웨어가 직접 사용하는 주소

### 논리 주소 (Logical Address)
- 프로세스가 사용하는 **가상의 주소**
- CPU가 생성하며, OS의 도움으로 물리 주소로 변환됨

```
[프로세스의 논리 주소] → [MMU 변환] → [실제 물리 주소]
```

> **MMU (Memory Management Unit)**: 논리 주소를 물리 주소로 변환해주는 하드웨어

---

## 메모리 관리의 목표
1. **보호**: 프로세스가 다른 프로세스의 메모리를 침범하지 못하도록
2. **효율**: 제한된 메모리를 최대한 효율적으로 활용
3. **공유**: 필요 시 여러 프로세스가 메모리를 공유 가능하게

---

## 메모리 단편화 (Fragmentation)

### 외부 단편화 (External Fragmentation)
- 남은 메모리 총량은 충분하지만, **연속된 공간이 없어** 할당 불가
```
[사용중][빈공간 2KB][사용중][빈공간 3KB][사용중]
→ 총 5KB 여유 있지만, 4KB 연속 공간 없음
```

### 내부 단편화 (Internal Fragmentation)
- 할당된 메모리 블록 내에 **사용하지 않는 공간** 발생
```
3KB 요청 → 4KB 블록 할당 → 1KB 낭비
```
