# 페이징 (Paging)

## 페이징이란?
**페이징(Paging)** 은 물리 메모리와 논리 메모리를 **고정된 크기의 블록**으로 나누어 관리하는 메모리 관리 기법입니다.

- **페이지(Page)**: 논리 메모리를 나눈 고정 크기 블록
- **프레임(Frame)**: 물리 메모리를 나눈 고정 크기 블록
- 페이지와 프레임의 크기는 동일 (보통 4KB)

---

## 페이징의 핵심 아이디어

프로세스의 논리 주소 공간을 **연속적이지 않아도** 물리 메모리 어디든 배치 가능합니다.

```
[논리 메모리]          [물리 메모리]
┌──────────┐          ┌──────────┐
│  페이지 0 │ ───────► │ 프레임 3 │
├──────────┤          ├──────────┤
│  페이지 1 │ ──┐      │ 프레임 0 │ (다른 프로세스)
├──────────┤   └────► │ 프레임 1 │ (다른 프로세스)
│  페이지 2 │ ──────► │ 프레임 2 │ (다른 프로세스)
└──────────┘          ├──────────┤
                      │ 프레임 4 │ ← 페이지 1 위치
                      ├──────────┤
                      │ 프레임 5 │ ← 페이지 2 위치
                      └──────────┘
```

→ 외부 단편화 없음! (프레임 단위로 빈틈없이 배치)

---

## 주소 변환 과정

논리 주소 = **페이지 번호(p)** + **페이지 내 오프셋(d)**

```
논리 주소: [페이지 번호 | 오프셋]
                ↓
          페이지 테이블 조회
                ↓
물리 주소: [프레임 번호  | 오프셋]
```

### 예시
- 페이지 크기: 4KB (2^12 = 4096 bytes)
- 논리 주소 12비트 = 페이지 번호 4비트 + 오프셋 8비트
- 페이지 번호 2 → 페이지 테이블에서 프레임 5 확인
- 물리 주소 = 프레임 5 시작 주소 + 오프셋

---

## 페이지 테이블 관리

### TLB (Translation Lookaside Buffer)
페이지 테이블 조회를 빠르게 하기 위한 **캐시**입니다.

```
CPU 논리 주소
     ↓
  TLB 조회 → TLB Hit  → 바로 물리 주소 변환 (빠름)
     ↓
  TLB Miss → 페이지 테이블 조회 → 물리 주소 변환 (느림)
              (TLB에 결과 저장)
```

## 페이징의 장단점

| 구분 | 내용 |
|------|------|
| **장점** | 외부 단편화 없음 |
| **장점** | 메모리 연속 배치 불필요 → 할당 유연 |
| **단점** | 내부 단편화 발생 가능 (마지막 페이지 남는 공간) |
| **단점** | 페이지 테이블 메모리 오버헤드 |
| **단점** | 주소 변환 시간 오버헤드 (TLB로 완화) |
