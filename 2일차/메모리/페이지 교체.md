# 페이지 교체

## 페이지 교체란?
물리 메모리(RAM)가 가득 찼을 때, **새 페이지를 올리기 위해 기존 페이지를 디스크로 내보내는 작업**입니다.
어떤 페이지를 내보낼지 결정하는 것이 **페이지 교체 알고리즘**입니다.

---

## 페이지 교체가 필요한 상황
```
① 새 페이지 필요 (Page Fault 발생)
② 빈 프레임 없음 (RAM 가득 참)
③ 교체할 페이지 선택 (피해자 페이지 선정)
④ 선택된 페이지를 디스크로 스왑 아웃
⑤ 새 페이지를 해당 프레임에 로드
```

---

## 페이지 교체 알고리즘

### 1. FIFO (First In First Out)
가장 **먼저 들어온 페이지를 먼저 교체**합니다.

```
페이지 참조 순서: 1 2 3 4 1 2 5
프레임 수: 3

[1][ ][ ] → [1][2][ ] → [1][2][3] → [4][2][3] → [4][1][3] → [4][1][2] → [5][1][2]
                                         ↑ 교체(1)      ↑ 교체(2)      ↑ 교체(3)      ↑ 교체(4)
```
- 구현이 단순함
- **Belady's Anomaly**: 프레임 수를 늘렸는데 오히려 페이지 폴트가 증가하는 현상

### 2. LRU (Least Recently Used)
**가장 오랫동안 사용되지 않은 페이지를 교체**합니다.

```
최근 사용 순서: ... → 3 → 1 → 2 (최근)
교체 대상: 3 (가장 오래된)
```
- **지역성(Locality)** 원리에 기반해 성능이 좋음
- 참조 시각을 기록해야 해서 구현 비용이 있음

### 3. OPT (Optimal)
**앞으로 가장 오랫동안 사용되지 않을 페이지를 교체**합니다.

- 이론적으로 **페이지 폴트 횟수가 가장 적음**
- **미래를 알아야 하므로 실제 구현 불가능** (성능 비교 기준으로만 사용)

### 4. LFU (Least Frequently Used)
**참조 횟수가 가장 적은 페이지를 교체**합니다.

- 자주 사용되는 페이지를 보호
- 과거에 많이 쓰였지만 현재는 안 쓰는 페이지가 남을 수 있음

---

## 알고리즘 비교

| 알고리즘 | 기준 | 구현 난이도 | 성능 |
|---------|------|-----------|------|
| FIFO | 입장 순서 | 쉬움 | 낮음 |
| LRU | 최근 사용 시간 | 중간 | 높음 |
| OPT | 미래 사용 여부 | 불가능 | 최적 |
| LFU | 사용 빈도 | 중간 | 중간 |

---

## 페이지 교체 시 고려 사항

### 수정 비트 (Dirty Bit)
- 페이지가 메모리에 올라온 후 **수정(write)** 되었는지 표시하는 비트
- Dirty = 1: 디스크와 내용이 다름 → **스왑 아웃 시 디스크에 써야 함**
- Dirty = 0: 디스크와 내용 동일 → **그냥 버리면 됨** (빠름)
