# 세그멘테이션 (Segmentation)

## 세그멘테이션이란?
**세그멘테이션(Segmentation)** 은 프로세스의 메모리를 **논리적인 단위(세그먼트)** 로 나누어 관리하는 메모리 기법입니다.

- **세그먼트(Segment)**: 코드, 데이터, 스택 등 **의미 있는 단위**로 나눈 블록
- 각 세그먼트는 **가변 크기** (페이징과의 차이점)

---

## 세그멘테이션의 핵심 아이디어

프로그램을 사람이 이해하는 논리적 단위로 분리합니다.

```
[프로세스 메모리]
┌──────────────────┐
│  코드 세그먼트    │ (크기: 3KB)
├──────────────────┤
│  데이터 세그먼트  │ (크기: 2KB)
├──────────────────┤
│  스택 세그먼트    │ (크기: 1KB)
├──────────────────┤
│  힙 세그먼트      │ (크기: 4KB)
└──────────────────┘
```

---

## 주소 변환 과정

논리 주소 = **세그먼트 번호(s)** + **세그먼트 내 오프셋(d)**

```
논리 주소: [세그먼트 번호 | 오프셋]
                ↓
         세그먼트 테이블 조회
         (base 주소, limit 크기 확인)
                ↓
  오프셋 < limit? → 물리 주소 = base + offset
  오프셋 >= limit? → 세그먼트 폴트! (접근 위반)
```

### 세그먼트 테이블 예시
| 세그먼트 번호 | 시작 주소(base) | 크기(limit) |
|------------|---------------|-----------|
| 0 (코드) | 1000 | 3000 |
| 1 (데이터) | 5000 | 2000 |
| 2 (스택) | 8000 | 1000 |

---

## 페이징 vs 세그멘테이션

| 구분 | 페이징 | 세그멘테이션 |
|------|--------|-----------|
| 분할 단위 | 고정 크기 (페이지) | 가변 크기 (논리적 단위) |
| 외부 단편화 | 없음 | 있음 |
| 내부 단편화 | 있음 (마지막 페이지) | 없음 |
| 보호 | 페이지 단위 | 세그먼트 단위 (더 직관적) |
| 공유 | 페이지 단위 공유 | 세그먼트 단위 공유 |
| 사용자 인식 | 인식 불가 | 논리 구조 반영 |

---

## 세그멘테이션의 장단점

| 구분 | 내용 |
|------|------|
| **장점** | 논리적 단위로 메모리 보호/공유 용이 |
| **장점** | 내부 단편화 없음 |
| **단점** | **외부 단편화** 발생 (가변 크기 때문) |
| **단점** | 메모리 압축(compaction) 작업 필요할 수 있음 |

---

## 세그멘테이션 + 페이징 (혼합 방식)
현대 OS는 두 방식을 혼합하여 사용합니다.

```
논리 주소
    ↓
세그먼테이션 (논리적 분리)
    ↓
페이징 (물리 메모리 매핑)
    ↓
물리 주소
```
→ 세그먼테이션의 **논리적 보호** + 페이징의 **외부 단편화 해결** 장점 결합
