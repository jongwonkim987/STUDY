# 가상 메모리 관리

## 가상 메모리란?
**가상 메모리(Virtual Memory)** 는 실제 물리 메모리보다 **더 큰 메모리를 사용하는 것처럼** 만들어주는 기술입니다.
실제 RAM이 부족해도, 디스크의 일부를 메모리처럼 활용합니다.

---

## 왜 필요한가?
- 물리 메모리(RAM)는 한정되어 있음
- 여러 프로그램을 동시에 실행하면 메모리 부족 가능
- 프로그램 전체를 메모리에 올리지 않아도 실행 가능하게 함

---

## 동작 원리

```
[프로세스의 가상 주소 공간]
┌────────────────────────┐
│  사용 중 → RAM에 존재   │
│  미사용  → 디스크에 존재│
└────────────────────────┘
          ↓
    페이지 테이블 조회
          ↓
┌──────────────────────────────────┐
│  RAM (물리 메모리)  │  디스크(스왑)│
│  일부 페이지만 존재 │  나머지 페이지│
└──────────────────────────────────┘
```

---

## 요구 페이징 (Demand Paging)
실제로 **필요한 페이지만** 메모리에 올리는 방식입니다.

1. 프로세스 실행 시작 → 최소한의 페이지만 메모리에 로드
2. 필요한 페이지가 메모리에 없으면 → **페이지 폴트(Page Fault)** 발생
3. OS가 디스크에서 해당 페이지를 메모리로 불러옴

### 페이지 폴트 처리 과정
```
① CPU가 가상 주소 접근
② 페이지 테이블 확인 → 해당 페이지 없음 (Page Fault!)
③ OS가 디스크에서 페이지를 찾아 빈 메모리 프레임에 로드
④ 페이지 테이블 업데이트
⑤ CPU가 다시 해당 주소 접근 → 정상 처리
```

---

## 스왑 공간 (Swap Space)
가상 메모리를 위해 디스크에 마련된 공간입니다.

- **스왑 인 (Swap In)**: 디스크 → RAM으로 페이지 이동
- **스왑 아웃 (Swap Out)**: RAM → 디스크로 페이지 이동

```
RAM 부족 시:
[잘 안 쓰는 페이지] → Swap Out → 디스크
[새 페이지 필요]    ← Swap In  ← 디스크
```

> 스왑이 잦으면 **스래싱(Thrashing)** 발생 → 성능 급격히 저하

---

## 스래싱 (Thrashing)
프로세스가 실제 작업보다 **페이지 교체에 더 많은 시간을 소비**하는 현상

```
메모리 부족 → 페이지 폴트 급증 → 스왑 빈번 → CPU 효율 급감
```

**해결 방법**: 물리 메모리 증설, 실행 프로세스 수 줄이기, 워킹 셋 모델 적용

---

## 페이지 테이블 (Page Table)
논리 주소(페이지 번호)를 물리 주소(프레임 번호)로 매핑하는 자료구조입니다.

| 페이지 번호 | 프레임 번호 | 유효 비트 |
|-----------|-----------|---------|
| 0 | 3 | 1 (메모리에 있음) |
| 1 | 7 | 1 (메모리에 있음) |
| 2 | - | 0 (디스크에 있음) |
