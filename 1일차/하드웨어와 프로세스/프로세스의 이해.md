# 프로세스의 이해

## 프로세스란?
**프로세스(Process)**는 실행 중인 프로그램을 의미합니다.
- 프로그램: 디스크에 저장된 정적인 코드 파일
- 프로세스: 메모리에 올라와 실행 중인 동적인 상태

## 프로세스 메모리 구조
```
높은 주소
┌──────────────┐
│   스택(Stack) │ ← 지역변수, 함수 호출 정보
├──────────────┤
│      ↓       │
│      ↑       │
├──────────────┤
│   힙(Heap)   │ ← 동적 할당 메모리
├──────────────┤
│   데이터(BSS)│ ← 초기화되지 않은 전역변수
├──────────────┤
│   데이터      │ ← 초기화된 전역변수
├──────────────┤
│   코드(Text) │ ← 실행 코드
└──────────────┘
낮은 주소
```

## PCB (Process Control Block)
운영체제가 프로세스를 관리하기 위해 유지하는 자료구조

| 항목 | 설명 |
|------|------|
| PID | 프로세스 식별자 |
| 프로세스 상태 | 실행, 준비, 대기 등 |
| PC | 다음 실행할 명령어 주소 |
| 레지스터 정보 | CPU 레지스터 값 |
| 메모리 정보 | 할당된 메모리 범위 |
| 입출력 정보 | 열린 파일 목록 |

## 프로세스 vs 프로그램
| 구분 | 프로그램 | 프로세스 |
|------|---------|---------|
| 상태 | 정적 | 동적 |
| 저장 위치 | 디스크 | 메모리 |
| 자원 | 없음 | CPU, 메모리 등 할당 |
