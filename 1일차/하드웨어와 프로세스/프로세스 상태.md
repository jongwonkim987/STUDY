# 프로세스 상태

## 프로세스 5가지 상태

### 1. 생성 (New / Created)
- 프로세스가 막 생성된 상태
- OS가 PCB를 생성하고 초기화

### 2. 준비 (Ready)
- CPU 할당을 기다리는 상태
- 실행에 필요한 모든 자원이 준비됨
- 스케줄러에 의해 선택되기를 기다림

### 3. 실행 (Running)
- CPU를 점유하여 명령어를 실행 중인 상태
- 단일 코어에서는 한 번에 하나의 프로세스만 실행 가능

### 4. 대기 (Waiting / Blocked)
- I/O 작업 등 특정 이벤트가 완료되기를 기다리는 상태
- CPU를 사용하지 않음

### 5. 종료 (Terminated / Exit)
- 프로세스 실행이 완료된 상태
- OS가 할당한 자원을 회수

## 상태 전이 다이어그램
```
         스케줄링
  준비 ──────────→ 실행
   ↑  ←──────────  ↓
   |    선점/타임아웃  |
   |                  ↓
   ← I/O 완료 ── 대기
                  ↑
              I/O 요청
```

## 상태 전이 정리
| 전이 | 원인 |
|------|------|
| 준비 → 실행 | 스케줄러가 CPU 할당 |
| 실행 → 준비 | 타임 아웃 또는 선점 |
| 실행 → 대기 | I/O 요청, 이벤트 대기 |
| 대기 → 준비 | I/O 완료, 이벤트 발생 |
| 실행 → 종료 | 실행 완료 또는 오류 |
