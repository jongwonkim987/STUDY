# 메시지 큐 (Message Queue)

## 메시지 큐란?
**메시지 큐(Message Queue)** 는 프로세스들이 **메시지(데이터 묶음)를 큐에 넣고 꺼내는 방식**으로 통신하는 IPC 메커니즘입니다.
커널이 메시지 큐를 관리하므로, 두 프로세스가 동시에 실행 중일 필요가 없습니다.

---

## 동작 원리

```
[프로세스 A]          [커널 메시지 큐]         [프로세스 B]
    │                ┌──────────────┐              │
    │  msgsnd()      │  메시지 3    │  msgrcv()    │
    │ ─────────────► │  메시지 2    │ ───────────► │
    │                │  메시지 1    │              │
    │                └──────────────┘              │
```

1. 송신 프로세스가 `msgsnd()`로 큐에 메시지 추가
2. 수신 프로세스가 `msgrcv()`로 큐에서 메시지 가져옴
3. 큐는 커널이 관리 → 프로세스가 종료되어도 메시지는 남아있음

---

## 메시지 구조
각 메시지는 **타입(type)** 과 **데이터(data)** 로 구성됩니다.

```c
struct message {
    long msg_type;   // 메시지 타입 (1 이상의 양수)
    char msg_data[256]; // 전송할 데이터
};
```

→ 수신자는 **특정 타입의 메시지만** 골라서 받을 수 있음

---

## 파이프와의 비교

| 항목 | 파이프 | 메시지 큐 |
|------|--------|---------|
| 데이터 단위 | 바이트 스트림 | 메시지 단위 |
| 방향 | 단방향 | 양방향 |
| 관계없는 프로세스 | 익명 파이프는 불가 | 가능 |
| 메시지 선택 | 순서대로만 | 타입별 선택 가능 |
| 데이터 유지 | 읽으면 사라짐 | 읽을 때까지 큐에 유지 |

---

## 장점과 단점

| 구분 | 내용 |
|------|------|
| **장점** | 메시지 단위 통신 → 경계가 명확 |
| **장점** | 비동기 통신 가능 (동시에 실행 안 해도 됨) |
| **장점** | 메시지 타입으로 우선순위 처리 가능 |
| **단점** | 커널을 경유하므로 공유 메모리보다 느림 |
| **단점** | 전송 가능한 메시지 크기에 제한 있음 |

---

## 활용 사례
- 작업 큐 (Task Queue): 웹 서버가 받은 요청을 워커 프로세스에 전달
- 분산 시스템의 메시지 브로커 (RabbitMQ, Kafka)
