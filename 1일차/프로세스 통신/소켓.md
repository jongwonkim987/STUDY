# 소켓 (Socket)

## 소켓이란?
**소켓(Socket)** 은 프로세스 간 통신을 위한 **양방향 통신 엔드포인트**입니다.
같은 컴퓨터 내 프로세스 사이뿐 아니라, **네트워크로 연결된 다른 컴퓨터의 프로세스**와도 통신할 수 있습니다.

---

## 소켓의 구조

```
[컴퓨터 A]                          [컴퓨터 B]
┌──────────────┐                  ┌──────────────┐
│  프로세스 A  │                  │  프로세스 B  │
│  ┌────────┐  │                  │  ┌────────┐  │
│  │소켓    │  │◄── 네트워크 ────►│  │소켓    │  │
│  └────────┘  │                  │  └────────┘  │
└──────────────┘                  └──────────────┘
  IP: 192.168.1.1                  IP: 192.168.1.2
  Port: 8080                        Port: 80
```

소켓은 **IP 주소 + 포트 번호**로 식별됩니다.

---

## 소켓 종류

### TCP 소켓 (스트림 소켓)
- **연결 지향**: 통신 전 연결(3-way handshake) 수립
- **신뢰성 보장**: 데이터 순서 보장, 유실 시 재전송
- 사용 예: HTTP, FTP, SSH

### UDP 소켓 (데이터그램 소켓)
- **비연결**: 연결 없이 바로 전송
- **빠르지만 신뢰성 없음**: 순서 보장 없음, 유실 가능
- 사용 예: 스트리밍, DNS, 게임

---

## 소켓 통신 흐름 (TCP 기준)

```
[서버]                          [클라이언트]
socket()   ← 소켓 생성 →        socket()
bind()     ← 주소 바인딩
listen()   ← 연결 대기
                  ←── connect() ─── 클라이언트가 연결 요청
accept()   ← 연결 수락 →
send/recv()  ←── 데이터 교환 ──►  send/recv()
close()    ← 연결 종료 →         close()
```

---

## 유닉스 도메인 소켓 (Unix Domain Socket)
같은 컴퓨터 내 프로세스 간 통신에 최적화된 소켓입니다.
- 네트워크 스택을 거치지 않아 **일반 소켓보다 빠름**
- 파일 시스템 경로로 식별 (예: `/tmp/mysql.sock`)
- 활용: MySQL, Nginx, PostgreSQL 등

---

## IPC 방식 비교 요약

| 방식 | 속도 | 범위 | 특징 |
|------|------|------|------|
| 공유 메모리 | 매우 빠름 | 같은 컴퓨터 | 동기화 필요 |
| 파이프 | 중간 | 같은 컴퓨터 | 단방향, 스트림 |
| 메시지 큐 | 중간 | 같은 컴퓨터 | 메시지 단위, 비동기 |
| 소켓 | 상대적으로 느림 | 네트워크 가능 | 양방향, 범용적 |
