# 프로세스 간 통신 (IPC)

## IPC란?
**IPC(Inter-Process Communication)** 는 서로 다른 프로세스가 데이터를 주고받는 방법입니다.

프로세스는 기본적으로 **독립된 메모리 공간**을 가지므로, 직접 다른 프로세스의 메모리에 접근할 수 없습니다.
OS가 제공하는 IPC 메커니즘을 통해 프로세스 간 통신이 가능합니다.

---

## 왜 IPC가 필요한가?
- 여러 프로세스가 협력하여 하나의 작업을 처리할 때
- 부모-자식 프로세스 간 데이터 전달
- 클라이언트-서버 구조에서 데이터 교환

---

## IPC 방식 분류

| 방식 | 설명 | 예시 |
|------|------|------|
| **공유 메모리** | 공통 메모리 영역을 통해 통신 | 실시간 데이터 공유 |
| **메시지 패싱** | 메시지를 주고받는 방식 | 메시지 큐, 파이프 |
| **소켓** | 네트워크를 통한 통신 | TCP/UDP 소켓 |
| **시그널** | 간단한 이벤트 알림 | kill 명령어 |
| **파이프** | 단방향 데이터 스트림 | 쉘의 `|` 연산자 |

---

## 통신 모델 비교

### 공유 메모리 모델
```
[프로세스 A] ←→ [공유 메모리 영역] ←→ [프로세스 B]
```
- 빠르지만 **동기화(synchronization)** 가 필요

### 메시지 패싱 모델
```
[프로세스 A] → send(msg) → [커널] → receive(msg) → [프로세스 B]
```
- 커널이 중간에서 중재 → 안전하지만 상대적으로 느림

---

## 동기화 문제
공유 메모리 방식에서는 두 프로세스가 동시에 같은 데이터를 수정하면 **경쟁 조건(Race Condition)** 이 발생할 수 있습니다.
→ **뮤텍스(Mutex)**, **세마포어(Semaphore)** 등으로 해결
