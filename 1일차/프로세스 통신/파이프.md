# 파이프 (Pipe)

## 파이프란?
파이프(Pipe)는 두 프로세스 사이에 **단방향 데이터 스트림**을 만드는 IPC 방식입니다.
한쪽 끝(write end)에서 데이터를 쓰면, 반대쪽 끝(read end)에서 읽을 수 있습니다.

---

## 파이프의 구조

```
[프로세스 A]  →  [파이프 버퍼]  →  [프로세스 B]
  (쓰기 end)     (커널 관리)       (읽기 end)
```

- **단방향(Unidirectional)**: 데이터는 한 방향으로만 흐름
- 양방향 통신이 필요하면 파이프 2개 사용

---

## 종류

### 익명 파이프 (Anonymous Pipe)
- **부모-자식 프로세스** 처럼 관련 있는 프로세스 사이에서만 사용 가능
- 프로세스 생성 시 함께 만들어짐
- 셸(shell)에서의 `|` 연산자가 대표적인 예

```bash
# 셸에서 파이프 사용 예시
ls -l | grep ".md"
# ls의 출력이 파이프를 통해 grep의 입력으로 전달됨
```

### 이름 있는 파이프 (Named Pipe / FIFO)
- 관련 없는 프로세스 사이에서도 사용 가능
- 파일 시스템에 파이프 파일로 존재
- 이름(파일 경로)으로 접근

```bash
# Named Pipe 생성
mkfifo my_pipe
# 쓰기
echo "hello" > my_pipe &
# 읽기
cat < my_pipe
```

---

## 특징

| 특징 | 내용 |
|------|------|
| 방향 | 단방향 (익명), 단방향/반이중 (이름 있는) |
| 속도 | 중간 (커널 버퍼 경유) |
| 용도 | 관련 프로세스 간 스트림 데이터 전송 |
| 저장 | 커널 내 임시 버퍼에 저장 |

---

## FIFO 특성
파이프는 **선입선출(FIFO)** 방식으로 동작합니다.
- 먼저 쓴 데이터가 먼저 읽힘
- 읽은 데이터는 파이프에서 사라짐
- 파이프가 가득 차면 쓰기 프로세스는 블록됨
- 파이프가 비어있으면 읽기 프로세스는 블록됨
