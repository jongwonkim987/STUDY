# 공유 메모리 (Shared Memory)

## 공유 메모리란?
두 개 이상의 프로세스가 **동일한 메모리 공간을 함께 사용**하는 IPC 방식입니다.
OS가 특정 메모리 영역을 만들어 여러 프로세스의 주소 공간에 매핑해줍니다.

---

## 동작 원리

```
[프로세스 A 메모리]        [프로세스 B 메모리]
┌───────────────┐         ┌───────────────┐
│  코드         │         │  코드         │
│  데이터       │         │  데이터       │
│  스택         │         │  스택         │
│  ┌─────────┐ │         │ ┌─────────┐  │
│  │공유 메모리│◄──────────►│공유 메모리│  │
│  └─────────┘ │         │ └─────────┘  │
└───────────────┘         └───────────────┘
        ↑                         ↑
        └──────── 같은 물리 메모리 ─┘
```

1. 프로세스 A가 OS에 공유 메모리 생성 요청
2. OS가 물리 메모리의 특정 영역을 공유 메모리로 지정
3. 프로세스 A, B 모두 해당 영역을 자신의 주소 공간에 매핑
4. 이후 두 프로세스가 직접 읽기/쓰기 가능

---

## 장점과 단점

| 구분 | 내용 |
|------|------|
| **장점** | 커널을 거치지 않아 **속도가 매우 빠름** |
| **장점** | 대용량 데이터 전송에 적합 |
| **단점** | 동시 접근 시 **동기화 문제** 발생 가능 |
| **단점** | 직접 동기화 처리를 해줘야 함 |

---

## 동기화 문제 예시

```
프로세스 A: 공유 변수 count 읽기 (값: 5)
프로세스 B: 공유 변수 count 읽기 (값: 5)
프로세스 A: count + 1 = 6 으로 저장
프로세스 B: count + 1 = 6 으로 저장 ← 결과가 7이어야 하는데 6!
```

→ 이를 **경쟁 조건(Race Condition)** 이라 하며, **뮤텍스/세마포어**로 해결

---

## 활용 사례
- 고성능이 필요한 프로세스 간 데이터 공유
- 데이터베이스 버퍼 공유
- 멀티프로세스 웹 서버 (nginx 등)
